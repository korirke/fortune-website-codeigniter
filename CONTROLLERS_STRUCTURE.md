# Controllers Structure

This document outlines all controllers that need to be created based on the Node.js backend structure.

## Controllers Created:
1. ✅ Auth/Auth.php - Authentication endpoints
2. ⏳ Public/Public.php - Public website content
3. ⏳ Public/Hero.php - Hero section
4. ⏳ Public/Search.php - Search functionality
5. ⏳ Public/Faq.php - FAQ public endpoints
6. ⏳ Public/About.php - About page public
7. ⏳ Public/Companies.php - Public company listings
8. ⏳ Public/Jobs.php - Public job listings
9. ⏳ Admin/Admin.php - Admin CMS endpoints
10. ⏳ Candidate/Candidate.php - Candidate profile management
11. ⏳ Jobs/Jobs.php - Job management
12. ⏳ Applications/Applications.php - Application management
13. ⏳ Companies/Companies.php - Company management
14. ⏳ Contact/Contact.php - Contact form
15. ⏳ Faq/Faq.php - FAQ admin
16. ⏳ Upload/Upload.php - File uploads
17. ⏳ Users/Users.php - User management
18. ⏳ AuditLog/AuditLog.php - Audit logs
19. ⏳ AdminCandidate/AdminCandidates.php - Admin candidate management
20. ⏳ RecruitmentAdmin/RecruitmentAdmin.php - Recruitment admin
21. ⏳ PricingRequest/PricingRequest.php - Quote requests
22. ⏳ About/About.php - About page admin
23. ⏳ Swagger/Swagger.php - Swagger documentation

All controllers should:
- Use Swagger/OpenAPI annotations
- Match Node.js endpoint structure exactly
- Use proper HTTP methods (GET, POST, PUT, DELETE, PATCH)
- Include proper authentication where needed
- Return consistent JSON responses
